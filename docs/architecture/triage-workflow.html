Triage Workflow — Step Diagram

Damian Galarza

Inside Station 1's triage-workflow, Slack Channels configured: two steps, no branching at the workflow level — the only loop lives inside the agent's own tool calls. Step 1 opens a fresh memory thread per run, lets the agent read the PR and its release notes, then a code-level guardrail checks the verdict before it can leave the step. Step 2 renders the card, posts it, and binds the thread so a later reply reuses this run's context.
